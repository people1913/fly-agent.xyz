VERIFICATION LAYER FOR AI ACQUISITION
验证AI带来的客户
是否真实产生了成交
Fly不是营销工具。不生成内容。不替代Agent。
Fly只验证一件事：AI推荐 → 用户咨询 → 最终成交之间是否存在真实、可信、可审计的关系。
类似支付需要Stripe验证资金流。AI时代，也需要验证：这次AI获客，到底有没有真正赚钱。
为什么需要验证层
AI正在替代传统流量入口。但AI获客存在一个根本问题：没人知道：成交是否真的来自AI。
传统广告有点击归因。但AI时代：
AI时代归因困境
多Agent协同
多轮对话
跨平台跳转
长周期成交
导致的后果
归因失真
佣金作弊
ROI失效
Fly的目标不是生成流量。而是：验证AI与成交之间，是否存在可信关系。
四步验证链路
Fly不是记录流程。Fly验证的是：每一次成交，是否真正来自该次AI推荐。
1

AI推荐

生成唯一Action ID作为验证入口凭证

2

身份与来源校验

验证咨询用户是否来自该次AI推荐

3

成交归因确认

确认成交与推荐之间的因果链路完整且可信

4

佣金与结果验证

归因成立，佣金结算才可执行。验证不通过，结算终止

Action ID 与验证维度
Action ID 只是入口凭证
每一次AI推荐都会生成唯一Action ID。Fly通过来源一致性校验、会话关联验证、归因完整性验证、哈希链审计，验证：这次成交，是否真正来自该次AI推荐。Action ID只是入口。验证成立，依赖完整的可信校验链路。
来源一致性校验
验证咨询用户与推荐来源的一致性
会话关联验证
验证推荐事件与咨询事件的会话关联
归因完整性验证
验证从推荐到成交的因果链路完整
