医美行业归因的特殊性
决策周期长（7-30天），用户可能跨多个平台反复咨询
跨平台触点分散：小红书种草、ChatGPT推荐、抖音广告、企业微信咨询

完整归因链路

1

AI种草

小红书/抖音博主发布医美内容，用户点击产生 Behavior ID

2

AI推荐

ChatGPT/AI搜索推荐机构，生成带 Action ID 的推荐链接

3

在线咨询

用户通过企业微信/在线客服咨询，生成咨询 Action ID

4

到院确认

用户到院面诊，生成 Verification ID，关联前序 Action ID

5

成交转化

用户完成消费，系统验证 Action ID → Verification ID 链路完整性

6

佣金结算

Trust Score ≥ 60 自动触发结算，争议单进入仲裁流程

Fly 六层验证

Gate 1
数据完整性
记录每个触点（种草/咨询/到院）的原始数据，确保归因链可追溯。任何节点数据丢失即触发告警。
Gate 2
安全策略
验证请求来源合法性：IP限流、UA检测、设备指纹。拦截刷单Bot和虚假到院请求。
Gate 3
归因链验证
验证 Action ID → Verification ID 的关联完整性。多触点归因时，按时间线分配权重。
Gate 4
部署验证
验证机构端SDK是否正确部署：短链跳转、回调接口、数据回传是否正常。
Gate 5
构建验证
验证归因规则配置正确：佣金比例、结算周期、争议规则是否符合合同约定。
Gate 6
健康验证
持续监控归因链路健康度：延迟、失败率、Trust Score 趋势。异常自动告警。
