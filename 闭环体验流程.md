# FLY智能体·闭环体验流程设计

> 让小老板觉得"这玩意真懂我"，不是"又一个工具"

---

## 一、闭环四步定义

| 步骤 | 目标 | 现有能力支撑 | 体验标准 |
|------|------|-------------|---------|
| **1. 记得住** | 第二次来认得他 | 扣子USER.md/MEMORY.md | "王姐，上次那套母亲节方案效果怎么样？" |
| **2. 出得全** | 文案+配图+数字人视频一体输出 | 文案生成+生图API+蝉镜API | 一次对话，三件套全齐 |
| **3. 发得出** | 复制粘贴就能发 | 标准化输出格式 | 零摩擦，不需再问"怎么用" |
| **4. 回得来** | 发完主动追问效果 | Bot人设追问机制 | "效果怎么样？给我说说" |

### 用户反馈闭环

```
方案发出后
     │
     ▼
FLY主动追问："效果怎么样？"
     │
     ├──▶ 用户说好 → 锁定这个风格，继续输出下一套
     │                "这套风格不错，继续？
     │                 我给你出一周7天的量。"
     │
     ├──▶ 用户说不好 → 快速调整，不重复被否的风格
     │                  "那换个方向，用对比冲击型？"
     │
     └──▶ 用户没回复 → 等他下次来先问效果
                      "上次那套发了没？评论区怎么样？"
```

## 三、记忆调用机制

### 3.1 记忆文件结构

**USER.md（用户画像）**
```markdown
## 基础信息
- 店铺名称：[从对话中提取]
- 店铺类型：[奶茶店/咖啡馆/服装店...]
- 目标客群：[年轻人/白领/宝妈...]
- 常用平台：[小红书/朋友圈/抖音...]
- 老板称呼：[王姐/张老板/阿雪...]

## 风格偏好
- 文案风格：[接地气/文艺/搞笑...]
- 视觉风格：[简约/潮流/真实感...]
- 品牌调性：[年轻活力/温馨家庭...]

## 雷区清单
- 不喜欢：[太官方/太正式/太长...]
```

**MEMORY.md（方案历史）**
```markdown
## 方案历史
### 方案001 - [日期]
- 方案名称：母亲节促销活动
- 使用场景：节日营销
- 方案亮点：温馨路线，突出感恩
- 渠道：小红书+朋友圈
- 文案风格：温情故事型
- 用户反馈：[待确认/满意/要修改]
- 效果数据：[用户补充]

### 方案002 - [日期]
- 方案名称：夏日新品推广
- 使用场景：产品上新
- 方案亮点：清凉感，强调新鲜
- 渠道：抖音
- 用户反馈：没回复
- 调整方向：换更抓眼球的钩子
```

### 3.2 记忆调用触发词

| 用户说什么 | Bot做什么 |
|-----------|----------|
| "我开XX店" | 首次建档USER.md |
| "上次那套" | 读取MEMORY.md调取历史方案 |
| "效果不错" | 更新MEMORY.md标记"满意" |
| "没人问" | 更新MEMORY.md标记"需调整" |
| 方案输出后 | 自动追加到MEMORY.md |

## 四、"出得全"执行标准

### 4.1 一体化输出流程

```
用户输入                      Bot响应
   │                            │
   ▼                            ▼
"我开奶茶店" ────────────▶ 识别行业
                            │
                            ▼
                        调用模板库
                        (奶茶店参数)
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
          朋友圈        小红书          抖音
           文案          文案           文案
              │             │             │
              ▼             ▼             ▼
          配图Prompt    配图Prompt    配图Prompt
              │             │             │
              ▼             ▼             ▼
          口播脚本(可选)     -              -
              │             │             │
              ▼             ▼             ▼
          数字人视频        -              -
              │             │             │
              └─────────────┼─────────────┘
                            │
                            ▼
                      结尾追问
                   "效果怎么样？"
```

### 4.2 各平台输出模板

**朋友圈输出格式：**
```
【朋友圈】
文案：[直接可发，3-5句话]
配图Prompt：[英文，50-100词]
发布时间：建议XX:XX
```

**小红书输出格式：**
```
【小红书】
标题：[2-3个备选]
正文：[可直接发布，含emoji和话题]
配图Prompt：[英文]
发布时间：建议XX:XX
```

**抖音输出格式：**
```
【抖音】
口播脚本：[可直接录制的文案，60秒内]
配图Prompt：[封面图英文描述]
数字人视频：[生成中/链接]
发布时间：建议XX:XX
```

## 五、体验对比

| 维度 | 闭环前 | 闭环后 |
|------|--------|--------|
| 第二次来 | "你是谁？我重新说一遍" | "阿雪，上次那套效果怎么样？" |
| 输出内容 | 文案给一张图给一个视频链接 | 一次性给全套，复制就发 |
| 效果追踪 | 用户不说就没了 | Bot主动追问，记录反馈 |
| 方案调整 | 每次从零开始 | 锁定好风格，避开被否的 |
| 平台扩展 | 只会一个平台 | 主动推荐没试过的 |

## 七、闭环成功标准

**让小老板说：**
- "这玩意真的记得我"

- "复制粘贴就发了，太方便"

**让小老板做：**
- 第二次回来
- 告诉效果反馈
- 尝试新平台
