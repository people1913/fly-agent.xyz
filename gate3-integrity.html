GATE 3 / 6
归因链验证
Attribution Chain Integrity Verification
从推荐到成交的归因链完整吗？哈希链有没有断裂？
为什么需要归因链验证
数据完整、来源真实，还不够。如果从推荐到成交的中间环节被篡改、删除或插入虚假事件，整条归因链的因果关系就不成立。归因链验证通过 SHA-256 哈希链确保每一个事件节点不可篡改、链式连续、全程可追溯。
没有归因链验证
中间事件可被悄悄删除
虚假事件可被插入归因链
时间戳可被回改伪造
归因因果关系无法自证
经过归因链验证
哈希链连续，任意删除 = 链断裂
任意篡改 = 后续哈希全部失效
Ed25519 签名防伪造
因果关系可独立验证
归因链验证做了什么
核心动作
Fly 为归因链上的每一个事件生成 SHA-256 哈希节点：hashₙ = SHA256(事件数据 + hashₙ₋₁)。从 CREATED 到当前状态，每个新哈希包含前一哈希，形成不可篡改的链式结构。任意修改一个字段 → 该哈希变化 → 后续所有哈希全部失效 → 链断裂。
✓ hash₁ = SHA256(CREATED 事件) → e3b0c4...
✓ hash₂ = SHA256(REFERRED 事件 + hash₁) → 7a8b9c...
✓ hash₃ = SHA256(CONVERTED 事件 + hash₂) → 1d2e3f...
✓ hash₄ = SHA256(SETTLED 事件 + hash₃) → 4a5b6c...
✗ 篡改 REFERRED 事件 → hash₂ 变化 → hash₃ hash₄ 全部失效 链断裂
STEP 1
哈希链连续性校验
验证从 CREATED 到当前状态，每个哈希节点的链接关系完整无断裂
STEP 2
事件完整性验证
重新计算每个事件的哈希值，与链上记录对比，确认无字段被篡改
STEP 3
数字签名验证
验证每个事件的 Ed25519 签名，确认事件由声称的 Agent 签发，不可伪造
