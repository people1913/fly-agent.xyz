Attribution

Verifiable Attribution Model
核心定义
Attribution = 可验证的归因
Fly 的归因不是"猜测谁贡献了转化"，而是"验证这个转化是否真实发生、来源是否可信、链路是否完整"。

传统归因 vs Fly 归因

传统归因模型
最后点击归因
算法黑箱，无法验证
各渠道数据不一致
依赖信任结算佣金
无法识别虚假归因
Fly 归因验证
全链路验证归因
哈希链可审计
单一数据源对齐
基于验证结果结算
虚假信号自动拦截

归因验证四阶段

阶段验证内容验证方式失败处理
推荐 推荐信号是否真实 验证来源渠道、推荐人身份 拒绝生成 Action ID
咨询 咨询是否由推荐触发 验证 Action ID 链路连续性 标记为异常信号
成交 交易是否归因正确 验证订单与 Action ID 绑定 拒绝绑定，人工审核
佣金 结算是否基于验证结果 验证全链路哈希链完整性 拒绝结算

归因数据模型

Action（动作）

归因的基本单元。每个 Action 包含 actionId、source（渠道）、referrer（推荐人）、status（验证状态）、hash（哈希值）。

Verification（验证记录）

对 Action 每个阶段的独立验证。每个 Verification 包含 actionId、stage（阶段）、result（结果）、timestamp（时间戳）。

Audit Event（审计事件）

链式审计记录。每个 Audit Event 包含前一个事件的哈希，形成不可篡改的审计链。

归因验证 API

POST /v1/agents

注册归因验证 Agent。Agent 是归因链路的参与方，每个 Agent 有独立的验证职责和权限。

POST /v1/audit/verify-chain

验证归因链完整性。提交 Action ID，系统返回全链路验证结果和哈希链状态。
