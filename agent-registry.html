Layer 2 · Agent Identity

Agent Registry

不是展示列表，是可验证索引。每个字段必须可校验、可审计、可治理、可追踪。未注册Agent无法通过Gateway进入生产环境。
Schema
可验证Registry字段定义
从"概念字段"修正为"基础设施字段"：每个字段必须有机器可验证语义。
以下字段对应 API 端点 POST /v1/agent/register 与 GET /v1/agent/:id，详见 API 文档。注册 POST /v1/agent/register · 查询 GET /v1/agent/:id
字段分组与5层架构映射： Identity / Runtime / Capability / Owner → Layer 1 Identity + Layer 5 Governance · Trust → Layer 2 Trust · Verification → Layer 3 Verification · Attribution 由 action_id + audit_log + attribution 字段共同实现，属于 Layer 4。
Identity · 身份
agent_idrequired
string · unique · immutable
全局唯一标识符，注册时生成，不可修改、不可伪造
fly-agent-7a3f2c1d
agent_fingerprintrequired
string · sha256 · verifiable
Runtime Hash指纹，用于验证Agent运行环境未被替换。Identity ≠ Name，必须可验证
sha256:e3b0c44298fc1c149afb...
修正：原Agent ID不可验证 → 补fingerprint
Runtime · 运行环境
runtime_providerrequired
enum · Coze / Dify / Claude / MCP
运行平台提供商，不是Runtime实例。Coze US和Coze CN是不同环境
Coze
runtime_envrequired
enum · production / sandbox / staging
运行环境标识。区分生产/沙盒/预发布，防止环境污染和Attribution错位
production
runtime_version
string · semver
Runtime版本号，用于审计和兼容性追踪
v1.2.0
修正：原Runtime=Coze语义错误 → 拆provider/env/version
Capability · 能力
capability_scoperequired
string · scoped
能力作用域，替代自然语言标签。声明+可验证
consult · lead_gen · data_analysis
tool_accessrequired
array · permission
工具访问权限白名单。声明的能力必须对应可执行的工具权限
["crm.read", "calendar.write"]
execution_moderequired
enum · autonomous / supervised / human-only
执行模式，限制Agent行为边界。不可治理的Capability = 没有Capability
supervised
修正：原Capability=sales_agent不可治理 → 拆scope/tool_access/mode
Owner · 归属
owner_orgrequired
string · org_id
归属组织标识，用于多租户隔离和权限控制
org_jiuyun_001
operator_idrequired
string · operator_ref
运营者ID，谁创建、谁维护、谁负责。不可审计的Owner = 没有Owner
op_zhangsan_001
修正：原Owner=fly不可审计 → 拆org/operator
Trust · 信任
trust_scorerequired
float · 0.0 — 10.0
综合信任评分，基于行为验证、审计记录等6维指标动态计算
8.5（示例值，实际由API计算）
trust_version
string · semver
评分算法版本，不可解释的Trust = 没有Trust
v1
trust_updated_atrequired
ISO8601 timestamp
评分最后更新时间。没有时间戳的评分不可审计
2026-06-04T16:30:00Z
修正：原Trust Score无来源 → 补trust_version + trust_updated_at，评分可审计
Verification · 验证
verification_statusrequired
enum · pending / partial / verified / failed / expired / revoked
验证是多值过程，不是二值标签。partial表示部分通过，expired需要重新验证
partial
verification_level
enum · L1 / L2 / L3 / L4 / L5
验证等级。L1=基础注册，L2=行为验证（6项行为检查通过），L3=生产准入（Trust Score≥7.0），L4=深度审计（全链路可追溯+安全合规通过），L5=最高信任（L4+持续运行30天无异常）
L2
verification_updated_atrequired
ISO8601 timestamp
验证状态最后更新时间
2026-06-04T16:30:00Z
修正：原Verification=Verified二值错误 → 多值状态+等级+时间

{
  // Identity
  "agent_id": "fly-agent-7a3f2c1d",
  "agent_fingerprint": "sha256:e3b0c44298fc1c149afbf4c8996fb924...",

  // Runtime
  "runtime_provider": "Coze",
  "runtime_env": "production",
  "runtime_version": "v1.2.0",

  // Capability
  "capability_scope": "consult",
  "tool_access": ["crm.read", "calendar.write"],
  "execution_mode": "supervised",

  // Owner
  "owner_org": "org_jiuyun_001",
  "operator_id": "op_zhangsan_001",

  // Trust
  "trust_score": 8.5,
  "trust_version": "v1",
  "trust_updated_at": "2026-06-04T16:30:00Z",

  // Verification
  "verification_status": "partial",
  "verification_level": "L2",
  "verification_updated_at": "2026-06-04T16:30:00Z",

}
Flow
验证流程：从pending到verified
1

注册 → status: pending, level: L1

提交agent_fingerprint + runtime三字段 + capability三字段 + owner双字段。系统生成agent_id，Trust Score=0.0

↓
2

行为验证 → status: partial, level: L2

沙盒运行，收集行为数据。验证fingerprint未变、tool_access未越权、execution_mode合规。Trust Score开始计算

↓
3

生产准入 → status: verified, level: L3（L4=全链路可追溯+安全合规通过，L5=L4+持续30天无异常）

Trust Score达标（≥7.0，由6维指标动态计算），所有字段可验证、可审计。允许进入生产环境

每个Agent，先验证再运行

不可验证的Registry = 没有Registry
