Action ID

Unique Attribution Identifier
定义
Action ID = 归因链路的唯一标识符
从推荐→咨询→成交→佣金，全链路共享同一个 Action ID，实现端到端追溯。每一条归因链路有且仅有一个 Action ID。

Action ID 的生命周期

1

生成：推荐触发

当用户通过推荐链接/二维码/渠道入口进入时，系统生成唯一的 Action ID，并记录来源渠道、推荐人、时间戳。

2

传递：咨询跟进

Action ID 随用户行为传递。咨询时系统验证 Action ID 的来源合法性，确认信号真实。

3

锁定：成交确认

交易完成时，Action ID 与订单绑定。验证链锁定归因结果，不可篡改。

4

结算：佣金分配

基于 Action ID 的验证链结果进行佣金结算。只有通过完整验证链的 Action ID 才能触发结算。

Action ID 的验证规则

GATE 1
来源验证：Action ID 必须由系统签发，不能伪造
GATE 2
时间验证：Action ID 的各阶段时间必须递增，不能回跳
GATE 3
完整性验证：Action ID 必须走完全链路才能触发结算
GATE 4
唯一性验证：同一个 Action ID 不能重复使用

API 接口

POST /v1/action

创建新的 Action ID。请求体需包含 source（来源渠道）和 referrer（推荐人标识）。返回唯一 Action ID 及初始验证状态。

GET /v1/action/:id

查询 Action ID 的完整生命周期。返回推荐→咨询→成交→佣金各阶段的验证状态和哈希值。
