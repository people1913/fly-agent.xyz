Action ID

Unique Attribution Identifier
定义
Action ID = 归因链路的唯一标识符
从推荐→咨询→成交→佣金，全链路共享同一个 Action ID，实现端到端追溯。每一条归因链路有且仅有一个 Action ID。

Action ID 是什么

一句话解释

Action ID 是 Fly 归因验证体系的核心原子单位。它就像一个"验证身份证"，从信号产生的第一刻起就被签发，伴随信号走过推荐→咨询→成交→佣金的全链路，每一步都记录验证结果和哈希值。

为什么需要 Action ID

传统归因的痛点：各环节各说各话。推荐平台说"我带来了100个客户"，咨询系统说"只有20个有效咨询"，成交系统说"只成交了3单"——数据对不上，谁也说不清问题在哪。Action ID 让每个环节用同一个ID说话，数据对齐一目了然。

生成规则

格式

Action ID 采用 act_{timestamp}_{random} 格式，例如：act_1717920000_a3f8b2c1

生成时机

当用户通过推荐链接/二维码/渠道入口进入时，系统在第一时间生成 Action ID。不是咨询时才生成，不是成交后才补——是信号产生的第一刻。

生成条件

必须满足以下条件才能生成：

① 来源渠道已验证（非伪造）

② 推荐人身份已确认（Agent注册有效）

③ 入口参数合法（防注入/防重放）

唯一性保证

全局唯一

每个 Action ID 在全系统中唯一，不会重复。基于时间戳 + 随机数 + 服务节点ID 三重保证。

不可复用

同一个 Action ID 只能走一次完整生命周期。一旦进入结算阶段，该 Action ID 永久锁定，不能再次触发任何归因流程。

防碰撞

即使在高并发场景下（1000 QPS），Action ID 的碰撞概率低于 10⁻¹⁸。设计上保证了跨节点、跨时间窗口的全局唯一性。

生命周期

1

生成：推荐触发 CREATED

用户通过推荐链接/二维码/渠道入口进入时，系统生成唯一的 Action ID，并记录来源渠道、推荐人、时间戳。此时验证链初始化，第一个哈希节点生成。

2

传递：咨询跟进 REFERRED

Action ID 随用户行为传递。咨询时系统验证 Action ID 的来源合法性，确认信号真实。验证通过则进入下一阶段，失败则标记异常。

3

锁定：成交确认 CONVERTED

交易完成时，Action ID 与订单绑定。验证链锁定归因结果，哈希链追加确认节点。此后归因结果不可篡改。

4

结算：佣金分配 SETTLED

基于 Action ID 的验证链结果进行佣金结算。只有通过完整验证链的 Action ID 才能触发结算。结算后 Action ID 永久锁定。

状态流转

状态含义可转入可转出
CREATED 已生成，等待推荐信号验证 — REFERRED / EXPIRED
REFERRED 推荐信号已验证，等待咨询 CREATED CONVERTED / ANOMALY
CONVERTED 成交已确认，等待结算 REFERRED SETTLED / DISPUTED
SETTLED 已结算，永久锁定 CONVERTED —（终态）
EXPIRED 超时未转化，已过期 CREATED —（终态）
ANOMALY 验证链异常，需人工审核 REFERRED CONVERTED / REJECTED
DISPUTED 归因争议中 CONVERTED SETTLED / REJECTED
REJECTED 验证失败，已拒绝 ANOMALY / DISPUTED —（终态）

验证规则

GATE 1
来源验证：Action ID 必须由系统签发，不能伪造
GATE 2
时间验证：Action ID 的各阶段时间必须递增，不能回跳
GATE 3
完整性验证：Action ID 必须走完全链路才能触发结算
GATE 4
唯一性验证：同一个 Action ID 不能重复使用

API 示例

POST /v1/action — 创建 Action ID
{
  "source": "douyin",
  "referrer": "agent_008",
  "entry_point": "qrcode",
  "metadata": {
    "campaign_id": "camp_2024_summer",
    "content_id": "video_12345"
  }
}
请求体必须包含 source（来源渠道）和 referrer（推荐人标识）。metadata 可选，用于记录活动ID、内容ID等附加信息。
响应 201 Created
{
  "action_id": "act_1717920000_a3f8b2c1",
  "status": "CREATED",
  "source": "douyin",
  "referrer": "agent_008",
  "hash": "sha256:e3b0c44298fc1c149afbf4c8996fb924...",
  "created_at": "2026-06-09T12:00:00Z",
  "expires_at": "2026-06-16T12:00:00Z"
}
返回唯一 Action ID、初始验证状态、初始哈希值和过期时间。默认7天未转化自动过期。
GET /v1/action/:id — 查询生命周期
{
  "action_id": "act_1717920000_a3f8b2c1",
  "status": "CONVERTED",
  "lifecycle": [
    {"stage": "CREATED",    "at": "2026-06-09T12:00:00Z", "hash": "sha256:e3b0c4..."},
    {"stage": "REFERRED",   "at": "2026-06-09T14:30:00Z", "hash": "sha256:7a8b9c..."},
    {"stage": "CONVERTED",  "at": "2026-06-10T09:15:00Z", "hash": "sha256:1d2e3f..."}
  ],
  "chain_valid": true,
  "trust_score": 87
}
返回 Action ID 的完整生命周期，包括每个阶段的时间、哈希值、链完整性判定和信任评分。
