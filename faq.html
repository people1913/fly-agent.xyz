FAQ

常见问题

关于 Fly

1Fly 是什么？ ▼
Fly 是 Agent Economy 的商业信任协议（Commercial Trust Protocol）。当多个 Agent 协作完成一笔商业交易时，Fly 通过 Claim → Evidence → Attestation → Trust Record 四层协议，为每个参与方的贡献提供可确定性重建的商业事实。Fly 不做支付，只证明结算条件已满足（Settlement Ready）。
2Fly 和传统归因工具有什么区别？ ▼
传统归因工具是"黑箱算法"，告诉你谁贡献了转化，但无法验证。Fly 的归因是白箱验证——每一步归因都经过独立验证，基于 Evidence Hash + Rule Hash 绑定，结果可追溯、可审计。核心公式：Evidence + Rule = Attribution。任何人 Replay 都得到完全一致的结果。
3什么是 Deterministic Replay？ ▼
Replay 是 Fly 协议的核心验证机制。给定相同的四个输入——Claim + Evidence + Rule + Protocol Version——任何人执行 Replay 都得到完全一致的结果。这意味着归因结论不依赖任何中心机构，任何独立方都可以验证。Replay 的输入永远只有这四个，不允许将中间产物（如 Attribution）作为输入。

技术问题

4四层协议分别是什么？ ▼
Layer 1: Claim（声明）——谁参与了交易？每个 Agent 声称自己做了什么。
Layer 2: Evidence（证据）——谁贡献了什么？可验证的数据来源、时间戳、行为记录。
Layer 3: Attestation（证明）——这可信吗？任何独立方均可执行 Replay 验证。
Layer 4: Trust Record（信任记录）——结算就绪。Trust Record 一旦生成不可修改，如需更新必须生成新版本。
5Rule 是如何保证不可伪造的？ ▼
Rule 是 Fly 协议中的一等对象（First-Class Object），包含 rule_id、issuer、version、hash、definition。核心验证：hash(rule.definition) == rule.hash。Attribution 绑定到 rule_hash 而非 rule 文本。Rule 必须可被公开发现和验证（Rule Registration），但协议不规定具体实现方式——可以是 HTTP、Git、DNS、DID 或 Blockchain。
6Settlement Ready 是什么意思？ ▼
Settlement Ready ≠ Settlement Executed。Fly 永远只做证明，不做执行。Fly 输出 Settlement Ready（证明结算条件已满足），但 Fly 不分钱。实际支付由参与方根据 Trust Record 自行完成。从本协议开始，所有文档、API、SDK 中统一使用 Settlement Ready 而非 Settlement。
7如何验证一份 Trust Report？ ▼
使用 Fly Verifier 对 Report 执行 10 项检查（v1.2）：schema 完整性、Claim 完整性、Evidence Hash 完整性、Rule 完整性、Attribution-Rule 绑定、Evidence-Claim 引用、Attribution 一致性、Settlement 正确性、Rule 注册验证、Issuer 信任评估。全部通过 → Report 可信。任何独立方都可以执行相同的验证。

接入与商务

8如何接入 Fly？ ▼
开发者可以通过 CTRS Schema 生成符合规范的 Trust Report，使用 Verifier 验证报告完整性，通过 Trust Loop 完成完整协议闭环。参考实现包括 Python Generator、Verifier、Trust Loop、Store 和 Registry。协议不绑定特定技术栈——PostgreSQL、S3、IPFS、SQLite 都可以作为存储实现。
9Fly 的定价模式是什么？ ▼
Fly 的商业模式是 Protocol Economy——商业价值发生以后收费，不是 SaaS 预付费。具体定价模型尚未冻结，将根据 Design Partner 反馈确定。核心原则：No Attribution, No Settlement. No Trust, No Payment.
10数据安全如何保障？ ▼
Fly 协议本身不存储任何原始数据。Evidence 通过 SHA-256 Hash 绑定，原始数据由参与方自行保管。Trust Report 只包含 Hash 引用和结构化元数据。协议不规定存储实现——你可以选择任何符合安全要求的存储方案。
