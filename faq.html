常见问题

关于 Fly

1Fly 是什么？ ▼
Fly 是归因验证基础设施（Attribution Verification Infrastructure）。它不是传统归因工具，而是一套让每一笔归因都可验证、可审计、可信任的基础设施。核心哲学是：Unverified Data ≠ Signal——未经验证的数据不叫 Signal。
2Fly 和传统归因工具有什么区别？ ▼
传统归因工具是"黑箱算法"，告诉你谁贡献了转化，但无法验证。Fly 的归因是"白箱验证"——每一步归因都经过独立验证，基于哈希链审计，结果可追溯、可审计。用户目标从"购买工具"变成"理解→验证→信任"。
3Signal 和 Data 有什么区别？ ▼
Signal = Verified State Transition（已验证状态跃迁）。只有通过验证链确认的数据才被称为 Signal，未经确认的只是 Data。Fly 的核心工作是验证 Data → Signal 的转换过程。

技术问题

4Action ID 是什么？ ▼
Action ID 是归因链路的唯一标识符。从推荐→咨询→成交→佣金，全链路共享同一个 Action ID。每一条归因链路有且仅有一个 Action ID，实现端到端追溯。通过 POST /v1/action 创建。
5哈希链是如何保证数据不可篡改的？ ▼
Fly 使用 SHA-256 哈希链：每个验证步骤的输出哈希作为下一步的输入。这意味着任何单点篡改都会导致后续所有哈希失效，整条链断裂。每日根哈希同步至公开账本，提供独立第三方验证能力。
6Fly 支持哪些渠道？ ▼
目前支持抖音、小红书、微信、美团、飞书等主流渠道。每个渠道有独立的接入适配层，确保不同渠道的信号格式统一转换为 Fly 标准验证格式。
7如何验证归因链的完整性？ ▼
通过 POST /v1/audit/verify-chain 接口提交 Action ID，系统会返回全链路验证结果，包括每个阶段的验证状态、哈希值和链完整性判定。也可以在沙盒验证页面可视化查看。

商务问题

8Fly 的定价模式是什么？ ▼
Fly 提供沙盒验证（免费）和企业版两种模式。沙盒验证允许你在测试环境中体验完整的归因验证流程；企业版提供生产环境部署、定制化验证规则和专属技术支持。具体方案请联系商务团队。
9数据安全如何保障？ ▼
所有数据传输使用 TLS 1.3 加密，静态数据采用 AES-256-GCM 加密，密钥由独立 KMS 管理并每 90 天轮换。访问控制基于角色的细粒度权限，确保每个用户只能访问其权限范围内的数据。详见 Trust Layer 和 Security Model。
