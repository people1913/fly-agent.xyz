Audit Trail 审计追踪

COMPLIANCE
Audit Trail
审计追踪链
每一步验证操作都留有不可篡改的审计记录
为什么需要审计追踪
归因验证的公信力来自可审计性。任何人（企业、审计方、监管方）都可以独立验证 Fly 的归因结果是否正确、数据是否被篡改。
审计验证方式
调用 GET /v1/audit/trail/:action_id 获取完整审计链，包含每个步骤的时间戳、操作者、输入哈希、输出哈希。任意步骤哈希不匹配即表示数据被篡改。
核心能力
核心能力
SHA-256 哈希链 + 每日根哈希公开，提供独立第三方验证能力
哈希链记录
每个验证步骤的输出哈希作为下一步输入，任何篡改导致链断裂
每日根哈希
每日根哈希同步至公开账本，提供时间锚点和独立验证
完整审计日志
记录所有 API 调用、验证操作、状态变更、数据访问，保留365天
第三方审计
支持独立审计方接入，提供只读审计 API 和验证工具
