Behavior ID 行为ID

Behavior ID
行为标识
这是什么
Behavior ID 是 Fly 为每一次 AI 推荐行为分配的唯一标识。它记录了这个推荐来自哪个 AI 平台、哪个 Agent、在什么时间发生。
核心理解

当用户在 ChatGPT、Claude、Coze 等 AI 平台上与 Agent 对话时，Agent 可能会推荐你的产品或服务。这次推荐行为本身需要被记录和追踪。

Behavior ID 就是这个记录的载体。它不是用户的身份标识，不是浏览器的 Cookie，也不是会话的 Session——它是推荐行为的标识。

一个 Behavior ID 包含：

  • 来源平台：推荐发生在哪个 AI 平台（ChatGPT、Claude、Coze、Dify 等）
  • Agent 标识：具体是哪个 AI Agent 发出的推荐（Agent ID 或名称）
  • 时间戳：推荐发生的精确时间
  • 推荐内容：Agent 推荐了什么产品、服务或链接
  • 上下文：用户问了什么问题、对话的背景

这些信息的组合构成了一个不可篡改的行为记录，为后续的归因验证提供基础。

为什么需要
AI 推荐了你的产品，但你不知道这条推荐从哪来。
没有 Behavior ID，每一次推荐都是「黑箱」——你知道流量来了，却不知道谁带来的。
与传统方案的区别

你可能在想：这不是 Cookie 或 UTM 参数能解决的吗？答案是：不能。

AI 时代的特殊挑战

AI 推荐与传统广告有本质区别：

  • 对话式推荐：用户不是在点击广告，而是在与 AI 对话，推荐是自然语言的一部分
  • 多轮交互：推荐可能发生在多轮对话之后，上下文复杂
  • 跨平台流动：用户可能在 ChatGPT 收到推荐，在 Claude 再次确认，最后通过 Coze 完成购买
  • Agent 自治：AI Agent 可能自主决定推荐什么，不受广告主直接控制

传统的归因工具无法处理这些场景。Behavior ID 专为 AI 推荐设计，能够准确记录每一次推荐行为的完整信息。

STEP 2

Behavior ID 的生命周期
从生成到归档
PHASE 1
生成
AI 推荐发生时，Fly SDK 自动生成 Behavior ID
PHASE 2
传播
Behavior ID 随推荐链接传递到用户浏览器
PHASE 3
关联
用户的后续行为（Action ID）与 Behavior ID 关联
PHASE 4
验证
Fly 验证行为链的完整性和真实性
PHASE 5
归档
验证完成后，记录长期保存用于审计
技术实现

Behavior ID 的生成遵循以下规则：

  • 唯一性：每个推荐行为生成一个全局唯一的 ID（UUID v4）
  • 不可预测：ID 本身不包含可推测的信息，防止伪造
  • 加密签名：使用 HMAC-SHA256 对关键信息签名，确保数据完整性
  • 时间绑定：ID 与精确时间戳绑定，支持时序分析

一个典型的 Behavior ID 示例：

BID-7f8a9b2c-3d4e-5f6a-7b8c-9d0e1f2a3b4c
平台: chatgpt
Agent: shopping-assistant
时间: 2026-06-17T10:30:45Z
签名: a1b2c3d4e5f6...

这个 ID 会被嵌入到推荐链接中，当用户点击时，Fly 的追踪系统会自动识别并记录。

验证价值
Behavior ID 带来的能力

有了 Behavior ID，你可以：

  • 精确归因：知道每一笔转化来自哪个 AI Agent 的推荐，不再"盲猜"
  • 公平结算：根据真实的推荐贡献计算佣金，避免争议
  • 优化策略：分析哪些 AI 平台、哪些 Agent 带来的转化质量最高
  • 防止欺诈：验证推荐行为的真实性，防止虚假归因
  • 合规审计：完整的推荐记录，满足监管和合规要求

更重要的是，Behavior ID 建立了 AI 推荐的可追溯性——这是 AI 商业化的基础设施。
