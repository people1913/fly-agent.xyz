Action ID
结果追踪 · 归因标识
Action ID 把「用户做了什么」和「谁推荐来的」关联起来
为什么需要 Action ID
Behavior ID 记录了推荐从哪来，但用户到底做了什么？这个付费是 AI 推荐带来的还是自然流量？Action ID 为每一个结果行为（注册、付费、下单）生成唯一标识，把推荐和行为关联起来。
Action ID 生命周期
CREATED（推荐触发）→ REFERRED（咨询验证）→ CONVERTED（成交确认）→ SETTLED（结算锁定）。异常路径：ANOMALY（验证异常）、DISPUTED（归因争议）、REJECTED（验证失败）。终态 SETTLED/REJECTED 不可逆。
核心定义
核心定义
Action ID = 归因链路的唯一标识符。从推荐→咨询→成交→佣金，全链路共享同一个 Action ID
唯一标识
格式 act_{timestamp}_{random}，碰撞概率低于 10⁻¹⁸，每条归因链路有且仅有一个
全生命周期
CREATED → REFERRED → CONVERTED → SETTLED，每个阶段记录时间戳和哈希
端到端追溯
一个 Action ID 串联推荐、咨询、成交、结算全流程，任何环节可查可验
终态锁定
SETTLED 后永久锁定，不可重复结算，防止同一 Action ID 多次触发佣金
