结果追踪
Action ID
归因标识
这是什么
Action ID 是 Fly 为用户的每一次结果行为（注册、付费、下单）分配的唯一标识。它把「用户做了什么」和「谁推荐来的」关联起来。
为什么需要
用户注册了、付费了，但这个收入归谁？
没有 Action ID，你只知道「有人付了钱」，不知道这个付费是 AI 推荐带来的还是自然流量。
怎么工作
STEP 1
用户行为发生
被推荐的用户完成了注册、付费、下单等关键动作
STEP 2
生成 Action ID
Fly 为这个结果行为生成唯一标识，绑定 Behavior ID
STEP 3
建立归因
Action ID 关联 Behavior ID，完成「推荐→结果」的归因链
下一步
Action ID 建立了归因关系，但「推荐→付费」的关联是真的吗？需要 Verification 来验证。
