什么是 Signal Flow
Signal Flow 是 Fly 验证层的信号流转路径。它展示了 Agent 行为数据如何从原始信号，经过6层 Gate 验证，最终产出可信的归因结果。理解信号流，就理解了 Fly 验证引擎的工作原理。
信号流转路径
信号生命周期
从行为发生到信任判定，信号经历采集、清洗、验证、归因四个阶段
1
行为采集 (Signal Collection)
Agent 行为触发后，系统采集原始信号：设备指纹、行为轨迹、时间戳
signal_type: action | click | conversion
2
信号清洗 (Signal Cleaning)
去重、去噪、标准化，过滤明显异常信号
dedup → normalize → validate_schema
3
Gate 验证 (Gate Verification)
通过 Gate 1-6 六层验证关卡，逐层过滤虚假行为
G1→G2→G3→G4→G5→G6
4
信任判定 (Trust Decision)
通过全部 Gate 的信号标记为 Verified，产出信任评分
status: verified | rejected | pending
6 层 Gate 验证详情
Gate 1 · 数据完整性
验证信号数据结构完整、字段合规、未被篡改
Gate 2 · 安全策略
检查来源安全、访问权限、无恶意行为特征
Gate 3 · 归因链
验证归因路径完整、因果关系可追溯、无断裂
Gate 4 · 部署验证
确认 Agent 部署环境合法、配置合规
Gate 5 · 构建验证
验证 Agent 构建产物完整、无供应链风险
Gate 6 · 健康验证
持续监控 Agent 运行健康状态、异常检测
