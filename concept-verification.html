为什么需要 Verification
Action ID 记录了「谁推荐→谁付费」的关联关系，但这个关联是真的吗？数据有没有被篡改？行为有没有被伪造？Verification 通过四重 Gate 验证，确保每一笔归因都是真实的、完整的、不可篡改的。
没有 Verification vs 经过 Verification
没有验证
  • 归因可能被伪造
  • 数据可能被篡改
  • 刷单无法识别
  • 佣金结算无依据
经过验证
  • 四重 Gate 层层过滤
  • 哈希链不可篡改
  • 异常自动标记 ANOMALY
  • 结算基于验证结果
验证失败会发生什么
任何一重 Gate 验证失败，该 Action ID 立即标记为 REJECTED（终态，不可逆）。系统记录失败原因、失败的 Gate 编号、原始数据哈希，供后续审计。验证失败的事件不会进入归因结算流程。
核心机制
核心机制
四重 Gate 验证：来源验证 → 因果验证 → 完整性验证 → 时间验证，层层过滤虚假数据
Gate 1 · 来源验证
确认信号来自声称的渠道——校验渠道回调签名、推荐人身份、渠道标识，伪造来源在第一步就被拦截
Gate 2 · 因果验证
确认事件之间的因果关系成立——推荐→咨询→成交的时序必须合理，间隔不能异常
Gate 3 · 完整性验证
确认哈希链无断裂——每个事件的 SHA-256 哈希包含前一事件的哈希，任何中间篡改都会导致链断裂
Gate 4 · 时间验证
确认事件时序合理——时间戳严格递增，无回跳、无异常间隔、无超时
