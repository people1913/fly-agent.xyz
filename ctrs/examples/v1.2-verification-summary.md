## 已知问题

### 1. Evidence Hash 序列化差异

**现象：** 3 条 evidence 的 `hash` 字段与使用标准 JSON 序列化（`sort_keys=True, separators=(',', ':')`）重新计算的 SHA-256 不一致。

**原因：** 报告生成时使用的 JSON 序列化方式与验证器不同（字段顺序、空格处理差异）。Rule hash 一致说明 Rule 部分使用了标准序列化。

**影响：** 不影响数据完整性和链路正确性，但无法通过自动化 Hash 校验。

**建议：** 统一生成器和验证器的序列化方式为 `json.dumps(data, sort_keys=True, separators=(',', ':'))`。

### 2. Rule 未注册

**现象：** `rule-commerce-multi-agent-v1` 未在 CTRS v1.2 Registry 中注册。

**原因：** 示例报告为手动生成，未通过 Trust Loop 流程运行。

**建议：** 通过 `ctrs-v1.2-trust-loop.py` 重新生成以完成注册。

### 3. Issuer 未信任

**现象：** `did:fly:commerce-platform` 信任等级为 `distrusted`。

**原因：** Issuer 未在信任评估体系中注册。

**建议：** 通过 Trust Loop 注册 Issuer 并建立信任链。
