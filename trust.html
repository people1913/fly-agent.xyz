Trust 信任层

定义
Trust Layer — 验证链的最终输出层
将通过验证的数据转化为可结算的 Signal，输出 Trust Score、Trust Proof 和结算就绪状态
Trust Layer 做什么

从 Data 到 Signal 的跃迁

Verification 确保数据真实，但不是所有真实数据都有相同的可信度。Trust Layer 根据验证历史、渠道质量、Agent 信誉综合评定信任等级，将 Data 跃迁为 Signal，进入结算流程。

Trust Score 计算

四维评分模型

Trust Score = 渠道基础分(30%) + Agent信誉分(25%) + 验证通过率(25%) + 历史异常率(20%)

评分可视化
默认阈值 60
0 · 高风险 40 · 预警线 60 · 结算线 80 · 高信任 100
≥ 60
结算就绪。4-Gate 全部通过 + Trust Score ≥ 阈值 → 可进入结算流程
< 40
风险预警。冻结相关结算，触发人工审核
≥ 80
高信任。优先结算 + 更高佣金比例 + 免担保
动态调整机制

每次验证事件都会更新评分

初始分数 60 分。每次成功验证 +0.5 分，每次异常 -5 分，每次 REJECTED -10 分。持续表现好则分数上升，异常则自动降级。

Trust Proof
TRUST PROOF · 信任证明

  "action_id": "act_1717920000_a3f8b2c1",
  "trust_score": 87,
  "gates_passed": ["GATE1", "GATE2", "GATE3", "GATE4"],
  "settlement_ready": true,
  "proof_hash": "h_8f2a3c...7d1e",
  "chain_depth": 6,
  "signed_by": "Ed25519:agent_01H8K9M2P",
  "timestamp": "2026-06-17T03:45:00Z"
}

Trust Proof 包含完整验证链摘要、Trust Score、结算就绪状态。使用 Ed25519 签名，任何字段被篡改 → 签名失效 → 可立即检测。

风险控制
HIGH · 立即冻结
Trust Score < 40 · 检测到哈希链断裂 · 多次 REJECTED · 签名验证失败
MEDIUM · 人工审核
Trust Score 40-59 · 异常率突增 · 单渠道集中度过高 · 时序异常
LOW · 持续监控
Trust Score 60-79 · 新 Agent 观察期 · 首次大额结算
争议仲裁

Dispute Resolution

合作方对归因结果提出异议时，提交完整 Verification Chain 证据链进行仲裁。Trust Proof + 哈希链记录作为不可篡改的仲裁依据，仲裁结果同样追加到哈希链，全程可追溯。

结算条件

Settlement Readiness

进入结算流程必须同时满足以下条件：

① 4-Gate 全部通过
② Trust Score ≥ 阈值（默认 60）
③ 哈希链完整无断裂
④ 数字签名全部验证通过
⑤ 无活跃争议（DISPUTED 状态未关闭）
