安全与合规

数据加密

Fly 对所有数据采用端到端加密策略，确保归因数据从生成到存储到结算全程受到保护。

传输加密

所有 API 调用与数据传输使用 TLS 1.3 加密，拒绝降级攻击。证书由权威 CA 签发，支持证书固定（Certificate Pinning）。

存储加密

静态数据采用 AES-256-GCM 加密，密钥由独立 KMS 管理，每 90 天自动轮换。密钥与数据物理隔离存储。

哈希链验证

每条 Action ID 的验证链路通过 SHA-256 哈希链连接，任何单点篡改都会导致整条链断裂，确保数据不可篡改。

链式结构

每个验证步骤的输出哈希作为下一步的输入，形成：推荐→咨询→成交→佣金的完整哈希链。任何中间节点的数据变更都会导致后续所有哈希失效。

链上锚点

每日将哈希链根哈希同步至公开账本，提供独立第三方验证能力。任何人都可以验证特定日期的数据完整性。

访问控制

基于角色的细粒度权限控制，确保每个用户只能访问其权限范围内的数据。

RBAC 权限模型

预设管理员、审计员、运营、只读四个角色，支持自定义角色与细粒度权限配置。所有权限变更留有审计日志。

API 安全

API 调用需携带有效 Token，Token 有效期 2 小时且支持即时吊销。所有 API 调用记录留存 180 天，支持回溯审查。

合规认证

Fly 遵循国内外数据安全与隐私保护法规，持续更新合规能力。

数据隐私

符合 GDPR 和《个人信息保护法》要求。用户数据最小化采集，支持数据导出与删除请求，数据处理协议可按需签署。

等保合规

系统架构满足等保二级要求，关键模块达到等保三级标准。数据存储于境内节点，不进行跨境数据传输。

安全审计

持续的安全审计与漏洞管理，保障系统长期安全运行。
